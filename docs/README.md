# Docs Index

Esta carpeta concentra documentación operativa y de referencia para que el root del repo quede reservado para código y configuración.

## Estructura

- `architecture/`: escalabilidad, arquitectura y capacidad.
- `audits/`: auditorías, seguridad, QA e investigación técnica.
- `demos/`: demos HTML y material de showcase.
- `ops/`: contexto operativo, implementación, monitoreo y setup técnico.
- `plans/`: planes de trabajo, QA y seguimiento.
- `reference/`: archivos de orientación rápida y "start here".
- `reports/`: reportes ejecutivos y resúmenes de avance.
- `seo/`: estrategia y materiales SEO.

## Queda en el root

Se dejaron arriba solo los archivos que funcionan como entrypoint real del proyecto o configuración crítica:

- `README.md`
- `CLAUDE.md`
- `CODEX.md`
- `DEPLOY.md`
- `package.json`
- `vite.config.ts`
- `firebase.json`
- `firestore.rules`
- `storage.rules`
- `capacitor.config.ts`
